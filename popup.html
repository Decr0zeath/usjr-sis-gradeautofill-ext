Ensure columns match the expected grade layout
First column: student_id; 2nd column: grade

Step-by-Step Guide

  1. Prepare your CSV file
    File must be in .csv format
    Make sure the CSV file contains only two (2) columns
    The 1st column should be named: student_id
    The 2nd column should be named: grade
    student_id should only contain the student's ID Number
    grade should be in the proper grading format (e.g., 3.0, 2.5, 1.3)
    Grades from 3.1 to 3.9, 5.0 or F are automatically set to 5.0
    Don't use grades 4.0 - 4.9, as the SIS will treat it as invalid
    You may grade F for a failed grade
    You may grade 6.0 or NC (all caps)
    You may grade 7.0 or INC (all caps)
  2. Select the course to be graded
    Go to the Post Grades page in the SIS
    Choose the course you want to apply grades to
  3. Upload the CSV file
    Click the upload option on the mainpage
    Select your prepared CSV file.
  4. Apply the Grades
    Click Apply Grades
    Review the applied grades for accuracy
    Pale Red means the grade in theCSV is INVALID
    Pale Yellow means the grade in the CSV is VALID
    Plain White means there is a bug in my code HAHAHA
  5. Save or submit the grades
    Saving/submitting is manual
    This ensures you can verify all grades before clicking Save or Submit
    Final submission still requires a human check
